Questão 5:

Diferenças Rápidas: innerHTML vs. innerText vs. textContent

  • innerHTML: O que faz: Acessa ou define todo o conteúdo HTML (tags + texto) dentro de um elemento. Ponto Chave: Interpreta as tags HTML. Cuidado: Usar para definir conteúdo de fontes não confiáveis pode ser um risco de segurança (XSS)
  • innerText: O que faz: Acessa ou define o texto visível ao usuário. Ponto Chave: Leva em conta o CSS (ignora texto oculto) e tenta simular a aparência renderizada. Pode ser mais lento.
  • textContent: O que faz: Acessa ou define todo o conteúdo de texto (incluindo o oculto), ignorando tags HTML. Ponto Chave: É a forma mais rápida e segura para manipular apenas texto, pois não analisa HTML nem CSS.
